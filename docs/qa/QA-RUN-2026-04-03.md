## Bugs Found

### Bug #1 — Admin Health Endpoint Returns 404

- **Severity:** Medium
- **Route:** `/admin`
- **API:** `GET /api/v1/admin/health`
- **Expected:** 200 with system health data
- **Actual:** 404 — endpoint does not exist
- **Console Error:** `[ApiClient] Error: GET /admin/health 404 Request failed with status code 404`
- **Impact:** Admin dashboard shows "System Degraded" with all services as "Unavailable" even though the app is functional. The health check panel is non-functional.
- **Reproduction:** Login as SUPER_ADMIN → navigate to `/admin` → observe System Health section

### Bug #2 — System Health Shows Degraded Despite Working App

- **Severity:** Low (related to Bug #1)
- **Route:** `/admin`
- **Expected:** System Health shows actual component status (DB: up, Redis: up, etc.)
- **Actual:** Shows DB: Unavailable, Redis: Unavailable, Kafka: Unavailable, Readiness: Unavailable. Only "Liveness State: Operational" is green.
- **Root Cause:** Health endpoint (Bug #1) returns 404, so the UI falls back to showing everything as unavailable.

### Bug #3 — QA Spec Mismatch: No Email/Password Login Form

- **Severity:** High (QA blocker — not an app bug, but a spec/implementation gap)
- **Route:** `/auth/login`
- **Expected (per UC-AUTH-001):** Email field, password field, "Sign In" button visible
- **Actual:** Only "Continue with Google" SSO button and "Demo Accounts" quick-login cards
- **Impact:** UC-AUTH-001 (email/password login), negative tests (wrong password, account lockout), and UC-AUTH-006 (password reset) are **all untestable** with the current UI
- **Recommendation:** Either update QA spec to reflect SSO-only login, or implement email/password form for non-Google-Workspace environments

### Bug #4 — Employee Profile Not Linked for Saran V

- **Severity:** Medium (data/seed issue)
- **Route:** `/me/dashboard` (logged in as Saran V)
- **Expected:** Dashboard shows employee-specific data (leave balance, attendance, etc.)
- **Actual:** "No Employee Profile Linked — Your account is not linked to an employee profile. Please contact your HR administrator."
- **Impact:** Self-service features (My Payslips, My Attendance, My Leaves) likely broken for this user
- **Root Cause:** User account `saran@nulogic.io` exists but is not linked to an employee record in the employees table
