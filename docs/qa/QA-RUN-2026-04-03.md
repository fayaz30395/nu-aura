## Bugs Found

### Bug #1 — React Hydration Mismatch in Sidebar and Login Page

- **Severity:** Low
- **Routes:** `/me/dashboard`, `/auth/login`
- **Type:** SSR Hydration Warning (dev-mode only)
- **Details:**
  - **Sidebar.tsx:500** — Server renders `className="w-full row-between ..."` but client renders
    `className="w-full flex items-center justify-between ..."`. The `row-between` utility class is
    expanded differently on client vs server.
  - **Login page (DemoLoginPanel)** — Lucide icon SVG `d` attribute mismatch between server and
    client render (mail icon rendered as users icon path).
- **Impact:** Dev-mode console errors only. Not visible to end users. Does not affect functionality.
  Would not appear in production build.
- **Recommendation:** Wrap dynamic icon selection in `useEffect` or `suppressHydrationWarning`, and
  ensure `row-between` utility is resolved identically on both server and client.

### Bug #2 — QA Spec Mismatch: No Email/Password Login Form

- **Severity:** High (QA blocker — not an app bug, but a spec/implementation gap)
- **Route:** `/auth/login`
- **Expected (per UC-AUTH-001):** Email field, password field, "Sign In" button visible
- **Actual:** Only "Continue with Google" SSO button and "Demo Accounts" quick-login cards (8 roles)
- **Impact:** UC-AUTH-001 (email/password login), negative tests (wrong password, account lockout),
  UC-AUTH-006 (password reset), UC-AUTH-003 (MFA setup) are **all untestable** with the current UI
- **Recommendation:** Either update QA spec to reflect SSO-only login, or implement email/password
  form for non-Google-Workspace environments

### Bug #3 — No Standalone Department Management Page

- **Severity:** Medium
- **Route:** `/departments` (Next.js error), `/admin/departments` (404)
- **Expected (per UC-DEPT-001):** Department list with CRUD operations (add, edit, delete
  departments)
- **Actual:** Department management is only available through the Organization Chart (
  `/admin/org-hierarchy`), which shows the reporting hierarchy but not a departments table/list with
  CRUD
- **Impact:** QA use case UC-DEPT-001 (department listing, filtering, create/edit/delete) cannot be
  fully tested. The API endpoint `GET /api/v1/departments/active` returns department data, but
  there's no frontend page to manage departments directly.
- **Recommendation:** Build a departments management page at `/admin/departments` or integrate
  department CRUD into the org hierarchy page

### Bug #4 — Admin Employee Stats Inconsistency

- **Severity:** Low (cosmetic / timing)
- **Route:** `/admin/employees`
- **Details:** On initial load, the stats cards showed "25 Total Employees, 10 Active, 0 Roles
  Defined, **0 Departments**". After a few seconds, Departments updated to **9**. The "0 Roles
  Defined" never updated despite roles being assigned to employees.
- **Impact:** Brief flash of incorrect data on admin page. "0 Roles Defined" appears permanently
  incorrect — roles (SUPER_ADMIN, MANAGER, TEAM_LEAD, EMPLOYEE) are clearly in use.
- **Recommendation:** Investigate the roles count endpoint — it may not be querying the right table,
  or the count refers to custom-defined roles (vs. system roles).

### Bug #5 — Login Page Console Errors After Logout

- **Severity:** Low
- **Route:** `/auth/login` (after sign out)
- **Details:** After signing out, the redirect to login page generates 401 errors on
  `/notifications/unread`, `/notifications/unread/count`, and `/workflow/inbox/count` because the
  Axios interceptor fires background requests before the auth state is fully cleared. The
  TokenRefreshManager then fires 3x `POST /auth/refresh` → 400.
- **Impact:** "3 errors" badge appears in the dev overlay on the login page. Not visible in
  production. Functionally correct — the session IS properly invalidated.
- **Recommendation:** Clear the React Query cache and cancel pending requests BEFORE redirecting to
  login after logout. This would prevent the orphaned 401s.
