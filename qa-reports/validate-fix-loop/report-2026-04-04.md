## Issues Found & Fixed

### FIX-001 (P0): `/admin/roles` — `permissions.reduce is not a function`
- **Affected Routes**: `/admin/roles`, `/admin/permissions`
- **Affected Roles**: ALL roles visiting admin pages
- **Root Cause**: `permissionsApi.getAllPermissions()` in `frontend/lib/api/roles.ts:75` returned a Spring Boot paginated response (`{content:[], pageable:{}, totalElements:349}`) but the calling code expected a flat `Permission[]` array
- **Fix**: Changed `getAllPermissions` to extract `.content` from the paginated response and pass `size=500` query param to get all permissions in one page
- **File**: `frontend/lib/api/roles.ts:75-79`
- **Before**:
```typescript
getAllPermissions: async (): Promise<Permission[]> => {
  const response = await apiClient.get<Permission[]>('/permissions');
  return response.data;
},
```
- **After**:
```typescript
getAllPermissions: async (): Promise<Permission[]> => {
  const response = await apiClient.get<{ content: Permission[]; totalElements: number }>('/permissions', {
    params: { size: 500 },
  });
  return response.data.content ?? [];
},
```
- **Recheck**: PASSED (hard refresh, 9 roles visible in table)

## RBAC Architecture Analysis

### Three-Layer Enforcement
1. **Middleware (Edge)**: Checks JWT cookie validity + expiry. SuperAdmin bypasses all. Redirects unauthenticated to `/auth/login`. **Does NOT check per-route permissions.**
2. **Client-Side (React)**: Sidebar filtering via `requiredPermission` on menu items. Page-level guards via `usePermissions()` hook redirect unauthorized users. Admin layout checks `isAdmin` flag.
3. **Backend API**: Returns 403 for unauthorized data access. `@RequiresPermission` annotation on controllers.

### Key Finding: No Middleware-Level Route Denial
All 67 routes return HTTP 200 for ANY authenticated user at the middleware level. RBAC enforcement is entirely client-side (Layer 2) and backend API-level (Layer 3). This is by design — the middleware is auth-only, not authz.
