## Fixes Applied

### Fix 1: LinkedIn service 403 handling (P1-1)
**File**: `frontend/lib/services/platform/linkedin.service.ts`
**Change**: Return empty result on 403 instead of throwing — EMPLOYEE role doesn't have PLATFORM:MANAGE permission for LinkedIn posts.

### Fix 2: Wall service 403 handling (P1-1)
**File**: `frontend/lib/services/core/wall.service.ts`
**Change**: Wrapped `getPosts()` in try/catch, return empty on 403.

### Fix 3: Analytics service 403 handling (P1-2)
**File**: `frontend/lib/services/core/analytics.service.ts`
**Change**: `getDashboardAnalytics()` now returns `null` on 403 instead of throwing.

### Fix 4: ApiClient 403 logging downgrade (P1-1, P1-2)
**File**: `frontend/lib/api/client.ts`
**Change**: GET requests returning 403 now logged as `warn` instead of `error` — these are expected for role-restricted resources.

### Fix 5: Wiki pages endpoint (P1-3)
**File**: `backend/src/main/java/com/hrms/api/knowledge/controller/WikiPageController.java`
**Change**: Added `@GetMapping` at root level to serve `GET /api/v1/knowledge/wiki/pages` — was returning 405 Method Not Allowed.

**File**: `backend/src/main/java/com/hrms/application/knowledge/service/WikiPageService.java`
**Change**: Added `getAllPages(Pageable)` method using existing `findByTenantId` repository method.

### Fix 6: Fluence service 403/405 handling (P1-3)
**File**: `frontend/lib/services/platform/fluence.service.ts`
**Change**: `listWikiPages()` and `listWikiSpaces()` now return empty on 403/405.
